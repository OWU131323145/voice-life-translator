# Voice Life Translator (VLT)

**「話すだけで、人生がデータになる」**
AI・センサー・リアルタイム通信を統合した、マルチデバイス対応ライフログアプリ

---

## 開発背景

日記や家計簿などのライフログは有用である一方、「入力の手間」によって継続が困難になるという課題があります。

本プロジェクトでは、

* **記録コストの極小化**
* **データの再利用価値の最大化**

を目的とし、「書く」という行為を以下に置き換えました：

* 音声 → AIによる文章化・構造化
* 画像 → Visionによる数値抽出
* 行動 → センサーによる自動計測

これにより、**無意識に近い形で蓄積されるライフログ基盤**を実現しています。

---

## ターゲットユーザー

* 記録が続かない人
* 健康・支出を可視化したい人
* 自分の行動データを分析したい人

---

## 🛠 技術的ハイライト

### 1. マルチモーダル入力 × LLM構造化

* Web Speech APIで音声取得
* OpenAI APIにより自然文へ整形＋JSON構造化
* Vision機能でレシート・体温などを自動抽出

**自由入力 → 構造化データへの変換を自動化**

---

### 2. 加速度センサーによる睡眠モニタリング

* `devicemotion` APIを使用
* 加速度の変化量を閾値ベースで検出
* 一定時間ごとの動きの頻度から睡眠状態をスコア化

**単なる睡眠時間ではなく「動き＝睡眠の質」を可視化**

---

### 3. SSEによるリアルタイム同期

* Server-Sent Events（SSE）を採用

**採用理由：**

* 一方向通信でユースケースを満たす

* WebSocketより実装がシンプル

* 接続管理コストが低い

* QRコードによるゼロ設定ペアリング

* PC ↔ スマホ間で即時同期

**軽量かつ実用的なリアルタイム同期を実現**

---

### 4. データ可視化と対話的分析

* p5.jsによるCanvas描画
* 支出・睡眠を時系列で可視化
* 過去ログをLLMに渡し対話分析

例：

* 「最近支出が多い項目は？」
* 「体調が悪い日の共通点は？」

**記録 → 分析 → 意思決定まで一貫**

## アーキテクチャ

### データフロー

```mermaid
graph LR
  A[音声 / 画像 / センサー] --> B[Frontend]
  B --> C[Node.js API]
  C --> D[OpenAI API]
  D --> C
  C --> E[Sync Layer (SSE)]
  E --> F[Storage / UI / Visualization]
```

---

### レイヤー設計

* **UI層**: 入力・表示・グラフ描画
* **API層**: LLM処理・データ整形
* **Sync層**: SSEによるリアルタイム同期
* **Storage層**: localStorageによる軽量保存

**責務分離により拡張性を担保**

---

## 📊 成果・価値

* 入力の手間を最小化（音声・画像・自動計測）
* 記録データを構造化し再利用可能に
* ユーザーが「意識せずに記録が溜まる」UXを実現

## セットアップ

```bash
npm install
```

### 2. 環境変数設定

`.env.example` をコピーして `.env` を作成し、APIキーを設定

### 3. 起動

```bash
node server.js
```
